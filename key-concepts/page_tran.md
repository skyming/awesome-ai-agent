### 📊 [思维链提示激发大型语言模型的推理能力 (Chain-of-Thought Prompting Elicits Reasoning in Large Language Models)](https://arxiv.org/pdf/2201.11903)

**摘要 (Abstract)**
> 本文探索了如何通过生成一系列中间推理步骤（我们称之为“思维链”，Chain-of-Thought）来显著提升大型语言模型进行复杂推理的能力。我们证明了，只要提供包含思维链的少样本提示（Few-shot prompting），足够大的语言模型就能展现出这种推理能力。我们在算术、常识和符号推理三大类基准测试上进行了实验。结果表明，思维链提示在超越特定规模的大模型上展现出了“涌现能力（Emergent property）”。例如，在 PaLM 540B 模型上使用仅包含 8 个示例的思维链提示，在 GSM8K 数学应用题基准测试中达到了当时的最先进（SOTA）水平，甚至超越了微调（fine-tuned）带有计算器的验证器模型。

#### 1. 引言 (Introduction)
语言模型在很多自然语言任务上表现出色，但在涉及多步数学计算或逻辑推理的任务上却经常碰壁。传统的少样本提示（Standard Few-shot Prompting）往往无法很好地处理这些复杂任务。

本文提出了一种简单而有效的方法：将“得出最终答案的中间思考过程”也加入到提示词中。这种方法的灵感来源于人类解决复杂问题时的认知过程——我们通常会把大问题拆解成几个小步骤来解决。我们将这种包含中间步骤的提示称为 **Chain-of-Thought (CoT)**。

#### 2. 思维链提示 (Chain-of-Thought Prompting)
在这一节中，作者详细定义了什么是思维链。

- **方法论**：如论文图1所示，标准提示直接给出问题和答案；而思维链提示在给出答案前，会展示一段文字推导过程。
- **四大优势**：
    - **分解复杂性**：允许模型将多步问题分解为中间步骤，这意味着可以将更多的计算资源分配给需要更多推理的问题。
    - **可解释性**：为模型的行为提供了一个可解释的窗口，表明模型是如何得出特定答案的，也方便人类查错。
    - **通用性**：不仅适用于数学题，还可以用于常识推理、符号操作等任何需要跨度思考的任务。
    - **易用性**：只需要极其少量的样本（比如 8 个示例），就能在大型现成模型上轻易激发这种能力，无需重新训练或微调模型。

#### 3. 算术推理实验 (Arithmetic Reasoning)
作者在 5 个数学应用题数据集（GSM8K, SVAMP, ASDiv, AQuA, MAWPS）上测试了 5 个不同的大模型（涵盖参数量从 4 亿到 5400 亿不等）。

- **核心发现 1：模型规模的“涌现”现象**。实验发现，思维链提示并不能改善小型模型（如小于 100B 参数）的表现。只有当模型规模足够大（如 PaLM 540B, GPT-3 175B）时，思维链提示才会带来突破性的性能提升。
- **核心发现 2：解决复杂问题更有效**。问题越复杂（需要更多步骤），思维链提示带来的性能提升就越显著。
- **结果**：PaLM 540B 结合思维链，在 GSM8K 测试集上准确率达到了 60.1%（当时的新纪录）。

#### 4. 常识推理实验 (Commonsense Reasoning)
除了数学，作者还测试了需要常识和物理直觉的任务（如 CSQA, StrategyQA, Date Understanding, Sports Understanding）。

- **现象**：对于 StrategyQA 和 Date Understanding，思维链提示同样大幅提高了模型的准确率。这证明了 CoT 是一种通用的逻辑强化手段。

#### 5. 符号推理实验 (Symbolic Reasoning)
作者设计了两个测试：
- **末字母拼接 (Last letter concatenation)**：比如问 "Elon Musk" 名字的最后字母拼接是什么（答案应该是 "nk"）。
- **硬币翻转 (Coin flip)**：比如“硬币正面朝上。张三翻转了它，李四没翻，王五翻转了它。现在是正面还是反面？”
- **结果**：面对这类需要过程追踪的符号任务，思维链提示让大模型以极高的准确率解决了这些问题，并且展现出了“长度泛化能力”。

#### 6. 讨论与局限性 (Discussion and Limitations)
- **无法保证完全正确**：模型生成的“思维链”虽然有助于推理，但有时也会产生错误的中间步骤（幻觉）。
- **推理代价**：生成长序列的思维链需要消耗更多的计算资源。
- **对小模型不友好**：它依赖于百亿/千亿级别大模型本身的预训练底座。

**总结 (Conclusion)**
思维链（CoT）证明了不需要改变模型架构或进行昂贵的微调训练，仅仅通过改变 Prompt，就能释放大型语言模型深藏的推理潜力。

### 📊 [思维树：使用大型语言模型进行深思熟虑的问题求解 (Tree of Thoughts: Deliberate Problem Solving with Large Language Models)](https://arxiv.org/abs/2305.10601)

**摘要 (Abstract)**
> 大型语言模型（LLMs）在解决各类任务上表现出色，但它们目前的推理机制主要依赖于从左到右、标记级别（token-level）的顺序决策过程。这类似于人类直觉式的、“系统1”的思考方式，而在需要探索、战略性前瞻或在最初决策失败时能够回溯的复杂任务上表现不佳。
> 为了克服这一限制，我们提出了一种新的语言模型推理框架——思维树（Tree of Thoughts, ToT）。它推广了经典的思维链（CoT）方法，通过探索多条推理路径来解决问题。ToT 将问题构建为对一棵“树”的搜索，树上的每个节点代表一个中间的“思维（Thought）”状态。通过结合语言模型的自我评估能力和经典搜索算法（如 BFS、DFS），ToT 允许模型进行深思熟虑的探索、向前看（Lookahead）以及回溯（Backtracking）。

#### 1. 引言 (Introduction)
双过程理论（Dual Process Theory）认为人类有两种认知模式：
- **系统 1**：快速、自动、无意识、直觉式。
- **系统 2**：缓慢、深思熟虑、有意识的规划。

ToT 的动机就是赋予 LLM“系统 2”的能力，允许模型进行多线思考、试错与全局规划。

#### 2. 背景与现有方法对比 (Background)
- **Input-Output (IO) Prompting**：标准提示，直接问答。
- **Chain-of-Thought (CoT)**：思维链，单一推理路径。
- **Self-Consistency with CoT (CoT-SC)**：思维链+自我一致性。生成多条路径投票，但路径间不互动。
- **Tree of Thoughts (ToT)**：维护一棵树，在中间节点生成多个分支并评估，有策略地选择最有希望的分支。

#### 3. 思维树框架核心组件 (Tree of Thoughts Framework)
- **思维分解 (Thought Decomposition)**：将复杂问题拆解为中间步骤。
- **思维生成器 (Thought Generator)**：
    - **Sample（采样）**：适用于思维空间大的任务（如写作）。
    - **Propose（提议）**：适用于思维空间受限的任务（如 24点）。
- **状态评估器 (State Evaluator)**：利用 LLM 自身作为裁判：
    - **Value（打分）**：给当前状态打分。
    - **Vote（投票）**：对比多个路径选出最佳。
- **搜索算法 (Search Algorithm)**：
    - **广度优先搜索 (BFS)**：齐头并进，保留最佳分支。
    - **深度优先搜索 (DFS)**：顺着一个方向探索，必要时回溯。

#### 4. 实验证明 (Experiments)
- **任务 1：24点游戏**：成功率从 CoT 的 4% 飙升至 **74%**。
- **任务 2：创意写作**：ToT 生成的文章连贯性远超标准提示。
- **任务 3：迷你填字游戏**：利用 DFS + 回溯机制，成功率显著提升。

#### 5. 讨论与局限性 (Discussion & Limitations)
- **优点**：结合启发式搜索与 LLM 能力，解决极高难度问题。
- **局限性**：计算成本高昂，消耗大量 Token。
- **结论**：适用于需要复杂逻辑、长线规划的高难度场景。

**总结**：
如果说 CoT 是一条道走到黑，ToT 就是在脑海中画出了一棵决策树，真正具备了类似人类的“深思熟虑”能力。

### 📊 [具有语言强化学习能力的语言智能体 (Reflexion: Language Agents with Verbal Reinforcement Learning)](https://arxiv.org/abs/2303.11366)

**摘要 (Abstract)**
> 本文提出了一种全新的框架——Reflexion（反思）。该框架不是通过更新模型权重来强化语言智能体，而是通过语言反馈（Linguistic Feedback）来实现。Reflexion 智能体会用自然语言对任务的反馈信号进行“口头”反思，并将反思文本保存在情节记忆缓冲区（Episodic Memory Buffer）中，以便在后续尝试中做出更好决策。

#### 1. 引言 (Introduction)
当前的 LLM 智能体往往会在同一个地方反复跌倒。Reflexion 将“标量奖励信号”转化为“丰富的语言反馈信号”，赋予智能体“吃一堑长一智”的自我反思与进化能力。

#### 2. Reflexion 框架的核心组件 (The Reflexion Framework)
- **Actor（行动者）**：负责生成文本和动作。
- **Evaluator（评估者）**：负责评价表现，给出反馈。
- **Self-Reflection 模型**：核心灵魂。在失败时介入，生成具体的“反思笔记”。
- **Memory（记忆缓冲区）**：存储反思笔记，作为下一次尝试的额外上下文。

#### 3. 实验证明 (Experiments)
- **任务 1：顺序决策 (AlfWorld)**：成功率从约 70% 提升至 **97%**。
- **任务 2：知识推理 (HotpotQA)**：通过反思纠正搜索方向，表现优异。
- **任务 3：代码生成 (HumanEval)**：GPT-4 准确率从 67% 提升至 **88%**。

#### 4. 讨论与优势 (Discussion & Advantages)
- **轻量且免微调**：利用上下文学习能力，无需更新参数。
- **语义梯度**：直接指出错误原因和改进方向。
- **可解释性强**：反思日志清晰展示认知演变过程。

**局限性 (Limitations)**：上下文窗口限制及可能产生的反思幻觉。

### 📊 [在语言模型中协同推理与行动 (ReAct: Synergizing Reasoning and Acting in Language Models)](https://arxiv.org/abs/2210.03629)

**摘要 (Abstract)**
> 本文提出了 ReAct 框架，旨在将大模型的推理（Reasoning）和行动（Acting）交织在一起。推理帮助行动（诱导、跟踪和更新计划），行动帮助推理（通过外部接口收集信息，缓解幻觉）。

#### 1. 引言 (Introduction)
ReAct 赋予了大模型“长出手和眼睛”的能力。它解决了纯推理（易产生幻觉）和纯行动（缺乏大局观）的割裂问题。

#### 2. ReAct 框架的核心范式 (The ReAct Framework)
采用 **Thought-Action-Observation** 循环：
- **Thought (思考)**：对当前状态进行推理分析。
- **Action (行动)**：选择并调用具体的工具/API。
- **Observation (观察)**：获取外部环境返回的结果。

#### 3. 实验证明与应用场景 (Experiments & Scenarios)
- **知识密集型推理**：通过搜索维基百科几乎根除了幻觉问题。
- **具身/决策任务**：引入 Thought 后，模型具备了状态追踪能力，成功率大幅提升。

#### 4. 讨论与深远影响 (Discussion & Impact)
ReAct 定义了现代 AI Agent 的工程标准。它不仅提高了准确率，还带来了极高的可解释性。

### 📊 [语言模型可以自学使用工具 (Toolformer: Language Models Can Teach Themselves to Use Tools)](https://arxiv.org/abs/2302.04761)

**摘要 (Abstract)**
> Toolformer 提出了一种让语言模型通过自监督（Self-supervised）的方式，自主决定何时调用 API、传递什么参数，以及如何将返回结果无缝融入后续生成中的方法。

#### 1. 引言 (Introduction)
让模型使用工具可以弥补其在算术、事实准确性和时效性上的短板。Toolformer 开启了大模型从“纯文本生成器”向“工具调用中控”转变的新范式。

#### 2. 核心方法：模型如何“自学”？ (The Approach)
- **采样 (Sample)**：在文本中尝试插入 API 调用指令。
- **执行并过滤 (Execute & Filter)**：执行 API 并根据 Loss 是否下降来过滤有价值的样本。
- **微调 (Finetune)**：使用保留的样例对模型进行微调。

#### 3. 实验配置与工具集 (Tools)
接入了计算器、问答系统、搜索引擎、翻译系统和日历等 5 个基础工具。

#### 4. 实验结果与意义 (Results & Significance)
- **降维打击**：6.7B 的 Toolformer 在多项测试中超越了 175B 的 GPT-3。
- **范式转移**：证明了模型可以通过自监督微调成为熟练操作 API 的“中控大脑”。
